### What's New in 1.8.0!

## **Features:**

### **New Heading Types & Path Planning**

- **Facing Point Heading:** A new `facingPoint` heading type lets a path segment continuously face a fixed coordinate on the field. Drag the target indicator anywhere on the canvas to aim the robot at a specific point throughout the segment. Fully supported in Java code export.
- **Universal Reverse Flag:** Every heading type — tangential, constant, linear, and the new facingPoint — now carries an optional reverse flag, allowing rear-facing travel on any heading mode with no configuration workarounds.
- **Preserve Heading on New Path:** When adding a new path segment (via the Add button, double-click on the field, the + button in the sequence list, keybindings, context menus, or the waypoint table), the heading type and relevant values are now automatically inherited from the previous segment. Linear paths carry forward their end angle as the new start angle; constant paths keep the same fixed angle; tangential paths preserve the reverse direction. No more manually re-selecting the heading type for every new segment.

### **Drivetrain Visualization**

- **Wheel Direction Arrows:** The robot overlay now renders arrows showing the individual wheel directions for holonomic and swerve drivetrains. For swerve, each module's steering angle is calculated from the robot's velocity vector and displayed live during animation. For holonomic, strafing vectors are shown to help visualize omni-wheel drive dynamics. Can be disabled in settings -> robot -> show wheel directions -> None.
- **Fake Heading Arrow Toggle:** A configurable overlay arrow now shows the robot's heading direction when using a custom robot image. You can enable it, choose its color, and it will always render at the correct orientation independent of the robot image's orientation.

### **File Manager Improvements**

- **Folder Support:** The File Manager now fully supports navigating into subfolders within your project directory. Browse and organize your `.pp` files across a nested directory structure without leaving the app.
- **Base Directory Constraint:** Navigation is now strictly constrained to the configured base directory — you can no longer accidentally browse above or outside your root folder.

### **Strategy Sheet**

- **Digital Strategy Notes:** Type your strategy notes directly in the Strategy Sheet Preview. Notes are saved with your project file and persist across sessions, replacing the old static printed lines. Physical blank lines still render if the notes area is left empty.

### **Section Looping**

- **Playback Section Looping:** Loop a specific subsection of your path during animation playback. Set a custom start and end point on the timeline and toggle looping on — now enabled by default. Previously known as "A/B Looping."

### **View Options & Toggles**

- **Quick-Access View Toggles:** New toggle buttons in the navbar let you instantly show or hide the Robot body, Onion Skin layers, and the Velocity Heatmap — no need to dig into the settings panel.

### **Keybindings**

- **Fine-Tuning Shortcuts:** New keybindings (`Shift` + `+` / `=` to increase, `Shift` + `-` / `_` to decrease) allow small-increment value adjustments for event markers, rotations, and wait times. Perfect for precise tuning without overriding existing controls.

### **Git Integration**

- **Live Git Status Updates:** Git status indicators in the File Manager and Navbar now refresh automatically on window focus — whenever you switch back from a terminal, VS Code, or another app. A lightweight dedicated IPC endpoint (`git:status`) runs only when needed, keeping resource usage minimal.

---

## **Enhancements & Optimizations:**

### **Performance**

- **Field Rendering Overhaul:** Significant performance improvements for lower-powered machines. Internal math has been optimized, heavy object cloning has been replaced with native operations, and redundant render passes have been eliminated. Animation playback and timeline scrubbing are now noticeably smoother.
- **Adaptive Sampling:** Path analysis now uses adaptive sampling, producing faster calculations and better visual fidelity especially on complex curves and long path sequences.

### **Animation Engine**

- **Auto-Reset on Change:** The animation controller now automatically seeks back to the beginning whenever your path is modified, ensuring previews always reflect the current state from the start.
- **Uninterrupted Playback:** Panning, zooming, or interacting with the canvas no longer pauses the animation. Playback continues uninterrupted while you adjust the view.

### **Default Robot Image**

- **Updated Default Robot:** The default robot image has been changed to better match a typical FTC robot profile out of the box. You can still customize or replace it freely from the Settings panel.

### **Visual Feedback**

- **Copy Code Confirmation:** The "Copy Code" button now briefly shows a "Copied!" message, giving immediate visual confirmation that code was placed on the clipboard.

---

## **Bug Fixes:**

- **Continuous Playback:** Interacting with the field canvas (panning, zooming, clicking UI controls) no longer inadvertently pauses animation playback.
- **Section Looping Reliability:** Loop range behavior has been stabilized across playback speed changes and edge-case timeline positions.
- **File Manager Navigation:** Navigation guards now correctly prevent escaping the configured base directory, even when using breadcrumb navigation or manual directory input.
- **Improved Accessibility:** Missing `aria-label` attributes added to icon-only buttons throughout, including the Section Looping toggle and navigation palette menu.
- **Usage Time Tracking:** Total usage time is now accurately tracked and persisted, ensuring the rating prompt no longer appears earlier than intended.
